Maîtrise de Git

Ce guide propose une approche professionnelle et structurée pour comprendre et maîtriser Git. Fini les erreurs de manipulation, apprenez à versionner votre code de manière fiable de A à Z.

Les fondations

Un dépôt Git est un espace isolé où l'historique complet de votre projet est méticuleusement consigné. Chaque révision majeure est enregistrée sous forme de commit, garantissant l'intégrité de votre code dans le temps.

Dépôt (Repository)

La base de données locale ou distante contenant vos fichiers et leur historique de modifications.

Commit

Une sauvegarde immuable capturant l'état exact du projet à un instant précis.

# Initialisation d'un nouveau dépôt Git
git init

# Vérification de l'état des fichiers modifiés
git status

Le Workflow

L'efficacité de Git repose sur une boucle de productivité simple. Il est crucial de séparer ses modifications logiquement via cette méthode en trois temps.

1

Modifier le code

Éditez ou ajoutez vos fichiers dans votre répertoire de travail classique.

2

Préparer la sauvegarde (Add)

Placez consciencieusement les fichiers prêts à être validés dans la zone d'attente, ou staging area.

3

Valider (Commit)

Figez l'ensemble des fichiers préparés en documentant la nature de la modification.

# Ajout des fichiers modifiés dans la zone de staging
git add .

# Création de la révision avec un ticket clair
git commit -m "feat: ajout du système de documentation"

Les Branches

L'isolation du travail est primordiale pour ne pas corrompre des environnements stables. Les branches vous permettent de développer plusieurs fonctionnalités en parallèle en toute sécurité.

Isolement

Créez autant de branches que de nouvelles fonctionnalités à développer ou de bugs à corriger.

Fusion

Une fois le code vérifié, intégrez le résultat final directement dans votre branche de production.

# Création et basculement sur une nouvelle branche cible
git switch -c feature/nouvelle-interface

# ... (Vos développements et commits) ...

# Retour sur la branche principale et fusion
git switch main
git merge feature/nouvelle-interface

Entraînement

Validation des acquis par la pratique :

  1. Ouvrez votre console et initialisez un répertoire vierge (git init).
  2. Générez un fichier README.md descriptif pour ce projet.
  3. Indexez le fichier produit avec git add README.md.
  4. Commitez la première itération (git commit -m "Initial commit").
  5. Consultez l'arbre d'historique (git log --oneline).
